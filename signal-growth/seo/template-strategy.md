# SEO And Template Strategy

Templates and comparison pages are acquisition surfaces, not only setup shortcuts.

## Technical Baseline

- `robots.txt` points crawlers at `/sitemap.xml`.
- Sitemap includes brand, proof, principles, press, dispatch, venues, venue demo, and comparison pages.
- Dispatch exposes RSS at `/changelog.rss`.
- Public schema describes Signal Studio as the organisation and software product.

## Priority Comparison Pages

Live under `/compare/[slug]`:

- `aisle-planner-alternative-ireland`
- `wedding-planning-workspace-for-venues`
- `notion-alternative-wedding-planners`
- `trello-alternative-builders`
- `project-management-students-no-sprints`

Rules:

- Compare by job, not by feature table.
- Say what Signal Studio refuses.
- Do not claim parity where it does not exist.
- Use the same tracking fields on every CTA: `source`, `campaign`, `audience`, `artifact`, `touch`, `venue`.

## Page Pattern

Each page should:

- solve one concrete problem
- show the workspace artifact clearly
- explain what the user gets
- make duplication obvious
- keep the broader Signal Studio loop visible
- route venues toward `/venues`, not a generic contact page
