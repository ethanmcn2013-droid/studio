# Venue Go-To-Market Execution Plan

Status: draft operating plan - 2026-05-26
Owner: founder
Boundary: no venue outreach is sent until the founder approves the visual pack, motion/video assets, and final email copy.

---

## Mission

Make the Venue Edition feel commercially inevitable for premium wedding venues before any outreach begins.

Signal Studio is not being sold as "planning software for weddings." It is the planning layer a premium venue gives every couple after booking so the couple feels looked after, the coordinator fields fewer repeated questions, and the venue looks more composed between deposit and wedding day.

## Recommended Sequence

| Phase | Timing | What happens | Output |
| --- | --- | --- | --- |
| 0. Freeze the strategy | Now | Keep the public routing contract stable and stop adding new wedding acquisition pages. | One model: venue-led is primary; self-serve is secondary. |
| 1. Build the pack | Before outreach | Draft the sales pack, implementation explanation, pricing framing, founder email, and follow-up. | Founder-ready copy deck and PDF outline. |
| 2. Build the proof | Before outreach | Upgrade the sample wedding demo into a narrated product story with motion/video notes. | Demo script and capture checklist. |
| 3. Instrument intent | Before outreach | Add or verify event tracking for venue page, demo page, CTA clicks, and template exits. | A lead-source map that separates venue and self-serve demand. |
| 4. Warm target prep | Before outreach | Finalize 30 accounts, sort into waves, identify likely owner/operator/coordinator paths. | Approved target board; no messages sent. |
| 5. First controlled send | Later, after founder approval | Send 10-12 highly personal messages, then learn before scaling. | Real venue responses and sharper offer language. |
| 6. Fulfilment rehearsal | Before first sale | Run a dry sale: invoice, code issue, redemption, welcome card, support handoff, renewal note. | Zero-surprise delivery path. |

## Targeting Logic

| Criterion | Why it matters |
| --- | --- |
| Exclusive-use or one-wedding-per-day model | The venue is selling calm, privacy, and service. Signal can extend that promise into planning. |
| Visible wedding team/coordinator | The buyer already knows planning friction exists and has staff who feel the pain. |
| Destination or multi-day weddings | Long planning windows, remote couples, guests, travel, suppliers, and repeated questions create a real coordination problem. |
| Premium brand and high average spend | EUR1,500-EUR4,000/year is easiest to justify when one incremental premium booking or smoother client experience covers it. |
| Independent or family-run decision path | Faster early adoption than large corporate procurement. |
| Existing content quality | Venues already investing in brand experience are more likely to value a polished post-booking layer. |

## No-Outreach Rule

Do not send venue emails until all of the following are approved:

| Gate | Required state |
| --- | --- |
| Sales pack | Founder approves the one-page PDF copy and final visual direction. |
| Demo story | Founder approves the "one wedding Tuesday" walkthrough and product captures. |
| Motion/video | Hero/motion/video assets are ready enough that the first impression feels premium. |
| Tracking | Venue page, demo page, CTA click, and target UTM checks are verified. |
| Ops rehearsal | One test venue code has been issued, redeemed, and confirmed end to end. |
| First batch | Founder approves the first 10-12 accounts and final message text. |
