# Cycle 4: Wedding/Events Proof Path

Cycle 4 turns the shared update artefact into a wedge-specific proof path.

The first proof URL is:

`/wedding-planning/update?source=roadmap_share&segment=weddings&role=viewer&campaign=founding_venue&artefact=shared_update`

## Purpose

Show the collaboration loop in a concrete wedding/events scene, not as an abstract suite idea.

A venue, planner, couple, or supplier should be able to open one shared update and understand:

- current planning state
- what is happening now
- what is held up
- what comes next
- what was recently decided

## Why This Matters

Weddings and events are coordination-heavy and naturally shareable. A clear planning update can travel from a venue coordinator to a couple, supplier, planner, or hotel team.

That makes it a strong early product-led growth surface.

## Current Implementation

Signal Timeline now includes bundled wedding planning proof data for `/wedding-planning/update`.

This is intentionally not a full live template yet. It proves the output format first.
