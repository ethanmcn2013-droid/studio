# Signal Studio Naming

This folder is the canonical source for Signal Studio product names, operating language, rename rules, and future-agent instructions.

Read in this order:

1. `LOCKED_OPERATING_VOCABULARY.md` - exact words to use for work, planning, release, and problem language.
2. `NAMING_CONSTITUTION.md` - durable naming rules and brand constraints.
3. `SIGNAL_STUDIO_LEXICON.md` - full vocabulary map.
4. `PRODUCT_LANGUAGE_GUIDE.md` - product surface language.
5. `INTERNAL_OPERATING_LANGUAGE.md` - internal operating language.
6. `DO_NOT_RENAME.md` - terms that must stay untouched.
7. `RENAME_MIGRATION_PLAN.md` - rename order and risk gates.
8. `LANGUAGE_UPDATE_PHASED_PLAN.md` - Initiative 01-08 execution plan.
9. `STRUCTURAL_RENAME_MAP.md` - code, route, asset, and config rename map.
10. `OPERATOR_GATED_RENAME_QUEUE.md` - repo/domain/env/database tasks left for operator action.
11. `AI_AGENT_INSTRUCTIONS.md` - agent instructions for future work.

Locked operating spine:

`Initiative -> Project -> Cycle -> Task -> Step`

Default replacements:

- Epic -> Project
- Sprint -> Cycle
- Ticket -> Task
- Bug -> Problem
- Backlog -> Queue
- Issue board -> Findings list
- Roadmap -> Timeline
- Analytics -> Signal

Use the old words only when quoting historic text, naming an external integration, preserving a stable route/API, or explicitly describing language Signal Studio refuses.
